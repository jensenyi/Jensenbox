# 创意车间 + 创意社区一次性落地边界

日期：2026-07-02
负责人：Jarvis
适用项目：Dachuan Growth OS / 内容工厂

## 一次性完成的定义

本次不是概念设计，而是完成一个本地可用的产品闭环：

1. 导航栏新增两个独立模块：`创意车间`、`创意社区`。
2. `创意车间`包含两个子模块：`文生图`、`文生视频`。
3. `创意社区`独立存在，用于热点帖子、评论、品牌案例、用户情绪和可转化选题分析。
4. 两个模块都必须能产生结构化结果，并能流转到后续模块。
5. 所有按钮必须有真实、诚实、可见的反馈。
6. 所有结果必须能沉淀到本地数据或资产库，不允许只做静态展示。

## 不在本次完成范围内

以下事项不得在本次强行实现：

1. 自动登录抖音、小红书、微博、Instagram、Facebook 等平台。
2. 读取主账号 Cookie 或绕过平台风控。
3. 自动评论、点赞、发布、私信、批量抓取。
4. 未配置合法 API Key 时假装完成真实图片/视频生成。
5. 新增后台常驻服务、定时任务、自动化任务或新线程。
6. 大规模重构现有内容工厂架构。
7. 新增不必要依赖。

## 创意车间

### 模块定位

创意车间是内容工厂的多模态生产模块，负责把文字内容转化为图片和视频成品，并进入多平台分发，最终沉淀到资产库。

### 子模块一：文生图

必须包含：

1. 文字输入区：标题、文案、脚本或选题。
2. 风格选择：潮流杂志、商业质感、强视觉海报、极简科技、品牌大片、街头视觉。
3. 适配平台：小红书、抖音、视频号、公众号、通用。
4. 用途选择：封面、配图、海报、长图、品牌视觉。
5. 专家脚本分析：
   - 核心观点是否清楚；
   - 情绪钩子是否明确；
   - 适合什么视觉表达；
   - 哪些表达太空；
   - 推荐 3 个视觉方向。
6. 成品区域：
   - 图片生成状态；
   - 图片预览占位或真实结果；
   - image prompt；
   - 平台适配建议；
   - 加入分发；
   - 保存到资产库。

大川已追加硬要求：创意车间不是方案生成器，必须接入图片生成能力并在右侧成品区域产出真实图片。

执行口径：

1. 文生图必须接入可用的 image 生成能力。
2. 成品区域固定放在右侧，显示真实生成图片、生成状态、失败原因、重新生成和保存入口。
3. Prompt 只能作为可查看的辅助信息，不能替代图片成品。
4. 若当前运行环境暂时无法调用 image 模型，必须停止并报告缺少的接入条件，不得用“Prompt 方案”冒充完成。

### 子模块二：文生视频

必须包含：

1. 文字输入区：文案、脚本、选题。
2. 风格选择：高能快剪、商业纪录片、科技感、街头潮流、知识型老板视角。
3. 适配平台：抖音、小红书、视频号、通用。
4. 视频参数：15s、30s、60s；口播、快闪、案例拆解、品牌观点、产品种草。
5. 专家脚本分析：
   - 前 3 秒钩子是否成立；
   - 节奏是否适合短视频；
   - 哪些段落应删减；
   - 需要哪些 B-roll；
   - 推荐视频结构。
6. 成品区域：
   - HyperFrames 视频方案；
   - 分镜结构；
   - 字幕重点；
   - 转场建议；
   - 成品预览或渲染状态；
   - 加入分发；
   - 保存到资产库。

大川已追加硬要求：文生视频必须接入 HyperFrames，并在右侧成品区域产出真实视频。

执行口径：

1. 文生视频必须接入 HyperFrames 生成/渲染链路。
2. 成品区域固定放在右侧，显示真实视频预览、渲染状态、失败原因、重新生成和保存入口。
3. 分镜和视频方案只能作为生成过程说明，不能替代视频成品。
4. 若当前运行环境暂时无法完成 HyperFrames 渲染，必须停止并报告缺少的接入条件，不得用“视频方案”冒充完成。

### 创意车间流转

文字输入
→ 专家脚本分析
→ 选择文生图/文生视频
→ 生成方案或成品
→ 加入多平台分发
→ 保存到资产库

## 创意社区

### 模块定位

创意社区是外部热点与用户反馈情报模块，负责收集品牌、抖音、小红书等方向的帖子、评论和案例，并转化为内容机会。

大川已进一步明确：创意社区不是手动输入表单，也不是简单导入工具；它应理解为类知乎、豆瓣、OpenAI Community 的内容社区/讨论池。用户打开后应看到已经聚合好的帖子、讨论、评论、热度和洞察，而不是先要求用户输入。

### 产品形态

创意社区必须呈现为社区信息流：

1. 顶部：频道筛选、来源筛选、刷新/同步状态。
2. 中部：帖子/讨论卡片流，类似知乎/豆瓣话题页。
3. 右侧或下方：热门关键词、用户情绪、品牌机会、可转化选题。
4. 卡片详情：摘要、代表评论、争议点、品牌机会、风险点、下一步动作。
5. 手动输入只能作为“补充采集”入口，不能是主界面核心。

### 推荐频道

第一版至少应有：

1. 品牌讨论；
2. 抖音热点；
3. 小红书种草；
4. 用户评论；
5. 营销案例；
6. AI/内容生产；
7. 消费趋势。

### 推荐公开源

可优先考虑合规公开源或搜索聚合：

1. 知乎公开问题/话题搜索结果；
2. 豆瓣公开小组/话题页面；
3. V2EX 公开话题；
4. 少数派公开文章；
5. B 站公开搜索/公开视频评论摘要；
6. 微博公开热搜/公开话题页；
7. 百度/必应/Exa 等搜索结果；
8. OpenAI Community / Reddit 等作为方法参考或 AI 社区信号源；
9. 小红书、抖音仅使用合规公开搜索、官方/授权数据源或明确可访问公开页，不读 Cookie，不绕风控。

若当前无法自动取得真实数据，必须显示“来源未接入/需授权/需公开源配置”，不得用静态假数据冒充自动抓取。

### 第一版社区目标调整为自动中文社区聚合

大川已追加硬要求：创意社区目标不是用户手动输入，而是系统自动抓取/聚合中文社区中关于品牌、抖音、小红书、热点内容和评论的信号，统一放入创意社区，并以类知乎/豆瓣/OpenAI Community 的信息流呈现。

执行口径：

1. 优先接入公开、合规、无需登录态的中文信息源。
2. 可采用搜索、RSS、公开网页、公开榜单、公开评论/讨论页、已批准工具或官方/合规 API。
3. 小红书、抖音等强登录/强风控平台不得读取主账号 Cookie，不得模拟异常行为，不得绕过平台限制。
4. 若某平台无法合规自动抓取，必须标注为“需人工授权/需合规数据源”，不能伪装为已接入。
5. 系统应支持自动聚合后统一结构化展示，同时保留手动导入作为补充入口。

创意社区仍建议保留手动导入：

1. 粘贴帖子内容；
2. 粘贴评论；
3. 粘贴链接；
4. 粘贴品牌案例；
5. 手动录入热点。

但手动导入不再是主流程，主流程应是自动聚合中文社区信号。

### 创意社区必须输出

每条内容必须结构化为：

1. 来源平台；
2. 内容摘要；
3. 用户情绪；
4. 关注原因；
5. 评论洞察；
6. 品牌机会；
7. 可转化选题；
8. 风险点；
9. 下一步动作。

### 创意社区按钮

每条卡片必须包含真实反馈：

1. 加入选题池；
2. 转入脚本工厂；
3. 转入创意车间；
4. 沉淀为资产。

若当前只是本地状态，必须显示为：`已加入本地选题池`，不得伪装为云端或平台写入。

## 风险筛查

### 平台与合规风险

1. 小红书、抖音等平台不做自动登录和规避风控。
2. 不读取主账号 Cookie。
3. 不做批量抓取和自动互动。
4. 对外部内容只做分析和转化建议，避免直接复制原文发布。
5. 自动聚合必须优先使用公开网页、RSS、搜索接口、官方 API 或用户明确批准的安全工具。
6. 对无法合规抓取的平台，展示“待接入/需授权/需合法数据源”，不得强行实现。

### 版权与素材风险

1. 外部帖子、评论、案例只能作为洞察来源。
2. 产出内容必须二次创作，不能原样搬运。
3. 图片和视频 prompt 不得要求复刻具体在世人物、品牌受保护素材或侵权视觉。

### 模型接入风险

1. 没有 image API 时，显示 prompt 方案，不显示假图片。
2. 没有 HyperFrames 渲染时，显示视频方案，不显示假成片。
3. API Key 不写入前端，不提交仓库，不展示在页面。

### 产品风险

1. 不做伪闭环。
2. 不做只会弹 toast 的按钮。
3. 不让创意社区和创意车间混在一个模块里。
4. 移动端必须可用。
5. 不牺牲现有总控台稳定性。

## 验收标准

本次完成后必须满足：

1. 导航能看到 `创意车间` 和 `创意社区` 两个独立入口。
2. 创意车间内能切换 `文生图` 和 `文生视频`。
3. 文生图/文生视频都有输入、风格、平台、专家分析、成品区。
4. 创意社区能自动聚合中文社区公开信号，并能手动导入帖子/评论/案例作为补充。
5. 创意社区能把洞察转入选题池、脚本工厂或创意车间。
6. 创意车间产出能进入多平台分发队列。
7. 创意车间产出能沉淀到资产库。
8. 所有按钮都有明确反馈。
9. 桌面和手机端均无明显重叠、横向溢出、空白页。
10. 不新增后台服务、不新增自动化、不新增线程。
11. 文生图右侧成品区必须显示真实图片结果。
12. 文生视频右侧成品区必须显示真实视频结果。

## 推荐执行顺序

1. 先加导航和路由。
2. 建立本地数据结构。
3. 做创意社区页面与中文公开源自动聚合，保留手动导入作为补充。
4. 做创意车间页面与文生图/文生视频子模块，右侧为真实成品区域。
5. 打通加入选题池、转入创意车间、加入分发、保存资产。
6. 做桌面和手机端验证。
7. 汇报改动文件、验证结果、风险和回滚方法。
