# MVP 验收标准

## 结构验收

- 项目目录、七个 Prompt、品牌档案、受众档案、规则和评分卡存在。
- 每日 Run 可通过 PowerShell 一键创建。
- Run 状态不能非法跳级。
- 三个模块线程有明确输入、输出和停止边界。

## 单次运行验收

- 输入包含 3 个热点、3 个竞品、1 个品牌案例。
- Research 输出 10 个非重复增长机会。
- Idea 生成 20 个完整选题并保留 Top 10。
- 最终选择 3 个选题，每个有完整脚本和视觉方案。
- Quality 每条提供 100 分评分、证据理由和明确结论。
- 80 分以下触发重写，最多 2 次。
- 至少形成 1 个带来源和更新时间的 GEO 资产。
- 发布表存在，但发布状态必须等待大川确认。

## 飞轮验收

- 发布数据能回写到 Review。
- 高价值问题进入 FAQ 或下一轮 Research。
- 重复低质量选题能被 rejected 记录阻止。
- 系统积累的是可复用资产，而非按日期堆积新闻。
