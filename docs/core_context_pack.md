# 内容工厂核心上下文包

## 固定控制台入口

- 地址：`http://127.0.0.1:4175/`
- 总控线程在涉及内容工厂操作、Run 状态或输入准备的回复中，优先保留一行：`[打开内容工厂](http://127.0.0.1:4175/)`。
- 控制台只负责输入、人工闸门、交接包和真实状态显示，不假装已经调用模型或完成发布。

这是所有模块的默认最小上下文。日常运行优先读本文件，不重复通读整套项目文档。

## 目标

Dachuan Growth OS 用内容推动业务增长，并持续沉淀趋势、内容、品牌、GEO 和增长资产。它不是新闻搬运器、AI 工具测评号或追求产量的内容机器。

## 大川定位

- 品牌增长操盘手、创业者、AI 时代增长实践者。
- 优先级：业务价值 > 增长价值 > 品牌资产价值 > 技术价值。
- 表达：有观点、有案例、有商业机制、有操盘视角、有执行建议；说人话，拒绝假大空。
- 审美：潮流、强视觉冲击，但视觉必须服务理解和品牌识别。

## MVP

- 输入：3 个热点、3 个竞品、1 个品牌案例，均需来源。
- 输出：10 个候选选题、3 条脚本、3 套视觉方案、1 份发布表、至少 1 份 GEO 资产。
- 流程：Research → Competitor → Idea → 人工确认 Top 3 → Script → Visual → Quality → GEO → 人工发布 → 真实数据复盘。

## 内容硬门

- 事实与观点分开，关键事实可回溯来源。
- 禁止纯资讯、虚构经历、伪造数据、复制竞品、标题不兑现和黑话堆砌。
- 视频通过四道创意门：记忆点、意外性、情绪弧、叙事变化。
- 质量分：90+ 直接通过；80-89 小修；70-79 重写；低于 70 废弃。最多两轮重写。
- 未通过质量门、未获大川确认，不得发布或执行外部动作。

## 能力分工

- 总控中台：状态、交接、资源、质量门和决策压缩。
- 情报与选题：Research、Competitor、Idea。
- 内容与视觉：Script、Visual。
- 质量与资产：Quality、GEO。
- 15 个 Agency 岗位是内部方法镜头，不是新线程。

## 成本纪律

- 无输入不运行，无明确用途不读取。
- 每阶段只读当前 Prompt、上一步输出和必要档案。
- 上游事实单点保存，下游用 ID 和文件路径引用。
- 原始影视知识和 Agency 岗位文件默认不加载。
- 不创建会议、日报、绩效巡检、空闲轮询或周期自动化。
- 先做最低成本样稿/样片，再决定是否增加投入。

## 最终汇报

只向大川提供：核心判断、Top 3、成品位置、主要风险、最多 3 个需确认动作。

## 主动权硬边界

- 可以主动，但按风险分级：低风险、可逆、低成本、围绕当前需求的小动作可以先做后报；中高风险动作必须先报备并等待大川确认。
- 可先做后报：只读定位、整理当前 Run、补齐状态说明、草稿级建议、风险提示、明显格式/错别字修正、阻止重复自动化或误配置继续扩散。
- 必须先确认：新功能、新 Run、新线程、新自动化、跨模块正式调度、规则改写、批量联网抓取、正式内容包、外部平台动作、登录/Cookie/账号相关操作。
- 不得自作主张扩展需求；但允许提出创造性方向，最多 3 条，等大川选择。
