# Module Ownership

Dachuan Growth OS can be operated by one person or by multiple specialized assistants. This document describes responsibility boundaries without relying on private thread IDs.

## Control Layer

Purpose:

- Define the content goal
- Decide what enters production
- Keep the workflow focused on useful outputs
- Approve publishing and major rule changes

Boundaries:

- Does not create unnecessary meetings or status rituals
- Does not duplicate work already owned by a module
- Does not publish externally without human approval

## Research And Ideas

Covers:

- Trend signals
- Competitor structures
- Brand cases
- User questions
- Topic opportunities

Output:

- Research packet
- Candidate topics
- Shortlisted topic recommendations

## Script Workshop

Covers:

- Short video scripts
- Xiaohongshu posts
- Video account scripts
- Hooks, titles, covers, comments, and platform-specific variants

Output:

- Publishable drafts
- Platform versions
- Script quality notes

## Creative Workshop

Covers:

- Text-to-image production
- Text-to-video production
- Reference image handling
- Visual style and platform adaptation

Output:

- Local image or video assets
- Creative prompts
- Distribution-ready asset records

## Quality And GEO Assets

Covers:

- Quality score
- Risk review
- Rewrite suggestions
- FAQ, glossary, method, case, and Q&A asset extraction

Output:

- Release checklist
- GEO assets
- Long-term content assets

## Operating Rules

- Work from files and artifact IDs, not long chat copies.
- Use the smallest module set needed for the current task.
- Stop after two rewrite attempts unless a specific blocker has been solved.
- Keep generated assets and private local state out of Git by default.
