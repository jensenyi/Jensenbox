# 内容生产硬规则

## 内容优先级

业务价值 > 增长价值 > 品牌资产价值 > 技术价值。

技术只有在改变成本、效率、体验、渠道或用户决策时才值得成为主题。

## 一条合格内容必须具备

1. 一个清晰观点，不做信息堆积。
2. 一个可信案例或证据。
3. 一个商业机制解释：为什么有效、影响什么。
4. 一个大川视角：小团队可以如何判断或复用。
5. 一个具体动作：今天、这周或这个月能做什么。
6. 来源和不确定性说明。

## 选题优先

- 新变化背后的旧问题发生了新解法。
- 国际品牌动作可翻译成中国市场或小团队方法。
- 用户有争议，但不是靠煽动制造争议。
- 能连接品牌、增长、GEO、用户决策或商业模式。
- 能沉淀 FAQ、词条、案例、模板或常青内容。

## 直接淘汰

- 纯新闻复述、纯 AI 资讯、纯工具清单。
- 只有情绪，没有事实和机制。
- 只有方法名词，没有适用条件。
- 与大川定位无关的泛娱乐热点。
- 无来源的“平台算法规律”。
- 为了日更而重复已有观点。

## 写作结构

优先使用：

```text
反常识/矛盾
→ 事实与案例
→ 商业机制
→ 大川判断
→ 可执行动作
```

## 语言要求

- 短句、具体、主动语态。
- 用“成本降低多少、减少哪一步、改变谁的决定”替代“赋能、破局、升维”。
- 不假装拥有未提供的数据和项目经历。
- 不虚构引用、用户评论、竞品表现和平台规则。
- 标题可以有张力，正文必须兑现。

## 平台改写规则

- 抖音：一个冲突，一个结论，一个动作；60 秒内完成。
- 小红书：可搜索、可收藏、可照着做；避免口播稿直接转图文。
- 视频号：老板视角、经营判断、信任感；不追求过度快节奏。
- 文章：完整解释机制，允许多案例和来源，适合 GEO 沉淀。

## 合规与品牌安全

- 广告、功效、收益和排名不得使用无证据绝对化表达。
- 引用品牌案例区分公开事实与个人分析。
- 图片、音乐、商标和用户内容注明来源及使用风险。
- 发布前必须经过大川确认；系统不自动发布。
