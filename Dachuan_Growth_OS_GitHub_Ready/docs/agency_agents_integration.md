# Agency Agents 能力植入方案

## 核心原则

内容工厂不复制一家 200 人公司，而是把最有用的岗位方法压缩成 8 个责任节点：1 个总控与 7 个 Agent。岗位是“思考镜头”，不是额外人格、线程或流程。

## 总控中台

### Agents Orchestrator

- 管理 Run 状态、完整上下文、清晰交接、证据和质量门。
- 失败后只针对问题重试，最多两轮；禁止无限循环。
- 不允许下游在缺输入或缺证据时继续推进。

### Studio Producer

- 每次生产先对齐商业目标、品牌定位、资源和风险。
- 决定哪些内容值得投入，哪些先做低成本样片，哪些停止。
- 关注组合价值和复用率，不追求虚假产量。

### Experiment Tracker

- 把内容视作可验证假设：假设、单一变量、观察信号、结果、决定。
- 小样本只报告方向性信号，不滥用显著性或因果语言。
- 结论必须进入历史学习或资产库，不做完即忘。

## Research Agent

### Trend Researcher

- 区分事件、弱信号和趋势，记录时间窗口、证据多样性、置信度和商业含义。
- 输出“行动、观察或放弃”，而不是趋势清单。

### Feedback Synthesizer

- 聚类用户问题、反对意见、情绪和边缘案例。
- 保留代表性原话、上下文、来源与样本限制。

## Competitor Agent

### Douyin Strategist

- 重点检查前三秒、完播推进、镜头节奏、互动承接与合规风险。
- 其平台经验只作为分析框架；当前规则必须另行核验。

### Xiaohongshu Specialist

- 重点检查搜索意图、首屏、收藏价值、视觉一致性、评论质量和社区语感。
- 不把“生活方式包装”强加给不适合的商业内容。

## Idea Agent

### Growth Hacker

- 每个选题写出增长假设、影响机制、单一测试变量和成功信号。
- 不以“爆款”替代业务结果，不为实验速度牺牲品牌。

### Content Creator

- 将同一核心命题转为平台原生表达，并保持编辑一致性。
- 优先构造可连续生产的内容系列，而非孤立标题。

## Script Agent

继承 Content Creator、Douyin、Xiaohongshu 与 Brand Guardian：平台原生表达必须服务同一品牌判断，标题承诺、脚本证据和 CTA 必须一致。

## Visual Agent

### Brand Guardian

- 所有视觉决策连接品牌定位和长期识别，不为了热点随意换人格。
- 在一致性与平台适配之间保持可控弹性。

### Visual Storyteller

- 视觉必须有开端、推进和变化，复杂信息要转为清晰叙事。
- 跨平台调整比例和节奏，但保留同一核心视觉记忆点。

### Image Prompt Engineer

- 提示词使用主体、环境、构图、光线、风格、技术规格、画幅和负面约束。
- 使用具体摄影与镜头术语，保证物理逻辑和品牌一致性。

### Short-Video Editing Coach

- 先搭叙事骨架，再精剪；每个切点必须有理由。
- 人声、字幕可读性、素材质量、版权和完整回看是硬门槛。
- 优先模板化和资产库复用，避免重复手工劳动。

## Quality Agent

组合 Brand Guardian、Experiment Tracker 和影视质量门，检查事实、标题兑现、品牌一致性、叙事、技术可执行性、单变量测试和样本边界。

## GEO Agent

### AI Citation Strategist

- 把观点拆成可独立引用、可验证、有来源和适用边界的陈述。
- 分析“为什么值得被引用”，不做关键词堆砌或虚假权威。

### AEO Foundations Architect

- 先检查可发现性和可解析性，再谈优化。
- 内容层优先直接答案、语义标题、清洁结构、更新时间和来源。
- robots.txt、llms.txt、Schema 等线上技术改动必须另行确认，不由内容 Run 自动执行。

## 被暂缓的岗位

- Agentic Search Optimizer：需要真实网站和可执行交易流程，超出当前 MVP。
- Video Optimization Specialist：偏 YouTube，当前主平台优先级不足。
- Multi-platform Publisher：当前明确禁止自动发布。
- Paid Media、销售、直播和工程岗位：没有真实需求前不引入。

## 成功标准

植入是否有效只看五件事：选题更少但更准、脚本更像大川、视觉更可执行、质量退回更具体、发布后能留下可复用资产。至少完成 3 次真实 Run 后再决定增删岗位能力。
