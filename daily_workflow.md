# 每日 MVP 工作流

执行前遵守 `docs/token_efficiency_protocol.md`。阶段按需串行启动，不允许七个 Agent 同时加载全部项目上下文。

## 0. 创建运行

执行 `tools/new-daily-run.ps1`。系统生成 `runs/YYYY-MM-DD/` 和 `run_manifest.json`。

## 1. 填写输入

在 `00_input.md` 中填写：

- 3 个热点：原始标题、链接、日期、为什么相关。
- 3 个竞品：账号/作者、平台、链接或原文、互动信息（若可得）。
- 1 个品牌案例：品牌、动作、公开来源、可借鉴问题。

不满足来源要求时，状态保持 `CREATED`。

## 2. Research Agent

读取 `prompts/01_research_agent.md`，产出 `01_research_packet.json`：

- 今日核心变化。
- 事实与来源。
- 10 个增长机会。
- 对大川的意义。
- 风险与不确定性。

通过标准：至少 7 个输入全部被处理，事实与观点分开。

完成后释放原始网页长文本，只把事实、短摘录、来源 URL 和判断写入结构化包供下游复用。

## 3. Competitor Agent

产出 `02_competitor_patterns.json`，拆解标题、钩子、结构、评论反馈、情绪触发、差异机会。不得把竞品表达直接复制进选题。

## 4. Idea Agent

完整版生成 20 个选题，MVP 最终保留 10 个。写入 `03_ideas.csv`。

选题先过硬筛选：非纯资讯、有明确观点、有商业价值、能形成行动或长期资产。

## 5. 选择 3 个生产选题

按观点、商业价值、人设匹配、平台潜力、资产价值预评分。默认选前三，大川可以改选。写入 `04_selected_ideas.json`。

## 6. Script Agent

每个选题选择一个主平台并生成对应完整脚本；同时给出其他平台改写建议。MVP 共 3 条完整内容。

## 7. Visual Agent

为三条内容分别产出封面标题、镜头、B-roll、视觉风格、图片提示词和字幕重点。

## 8. Quality Agent

使用 `content_scorecard.md` 逐项评分并检查一票否决项。

- 90+：通过。
- 80-89：小修。
- 70-79：重写。
- <70：废弃。

质量检查以 manifest 和目标内容为入口；只在发现来源疑点时回溯 Research，不默认重读所有原始输入。

## 9. GEO Agent

只处理 QA 通过内容，至少沉淀一种资产：FAQ、品牌词条、行业词条、方法论、案例卡、问答库或 GEO 长文。

## 10. 发布表

写入 `publish_calendar.csv`：日期、平台、内容 ID、标题、状态、负责人、素材位置、CTA、发布链接。

发布状态必须等待大川确认，系统不自动发平台。

## 11. 数据复盘

发布后填写真实数据和高价值评论。重点不是单次播放量，而是：

- 哪个问题最能引发目标受众回应？
- 哪个观点形成收藏、转发、私信或业务讨论？
- 哪个结构可以复用？
- 哪些 FAQ 和关键词应进入 GEO 资产？

复盘结论写入下一轮 Research 的 `historical_learnings`。

每次复盘只形成三类结论：继续复用、修改后再测、停止投入。没有足够样本时写“方向性信号”，不得宣称因果或统计显著。
